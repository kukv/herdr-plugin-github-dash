# CI 導入 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pull Request 上で fmt/lint・test・カバレッジを自動チェックする GitHub Actions CI を導入する。

**Architecture:** 1つのワークフロー `ci.yaml`（`on: pull_request`）に、独立した2ジョブ `lint` / `test` を並列で持たせる。lint は golangci-lint v2 で `run`（静的解析）と `fmt --diff`（フォーマット崩れ検出）を実行。test は gotestsum でテスト＋カバレッジプロファイルを生成し、octocov でカバレッジをPRコメント報告＋閾値判定する。CI をグリーンにするため、既存コードの実質的な lint 指摘5件を先に潰し、価値の低い指摘はルール抑制で対処する。

**Tech Stack:** GitHub Actions / golangci-lint v2.12.2 / gotestsum v1.13.0 / octocov (k1LoW/octocov-action@v1) / Go 1.26

## Global Constraints

- トリガーは `pull_request` のみ（main への直接 push では走らせない）。
- lint と test は別ジョブ・並列（`needs` で直列化しない）。
- Go モジュールの依存キャッシュは設定しない（`actions/setup-go` は `cache: false`）。
- ランナーは `ubuntu-latest`。Go バージョンは `go-version-file: go.mod` で解決。
- golangci-lint の設定は「実質重視」: 実質的な linter は有効に保ち、ノイズ（revive `exported`/`unused-parameter`、gosec `G204`、テストコードの gosec）はルール抑制で対処する。
- スコープ外: リリース（タグ/GitHub Release）自動化、version 一致ガード。
- 設計ドキュメント: `docs/superpowers/specs/2026-07-12-ci-setup-design.md`。

## Task 2: octocov 設定 + CI ワークフロー

カバレッジ設定と CI ワークフローを追加する。ワークフローの実挙動（2ジョブ並列・octocov のPRコメント）は PR を開いて GitHub 上で検証する。

**Files:**
- Create: `.octocov.yml`
- Create: `.github/workflows/ci.yaml`

**Interfaces:**
- Consumes: Task 1 で作成した `.golangci.yml`（lint ジョブが参照）。
- `.octocov.yml` は test ジョブが生成する `coverage.out` を参照する。

- [ ] **Step 1: `.octocov.yml` を作成**

```yaml
coverage:
  paths:
    - coverage.out
  acceptable: 80%
comment:
  if: is_pull_request
```

- [ ] **Step 2: `.github/workflows/ci.yaml` を作成**

```yaml
name: CI

on:
  pull_request:

permissions:
  contents: read
  pull-requests: write
  actions: read

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: false
      - name: Install golangci-lint
        run: curl -sSfL https://raw.githubusercontent.com/golangci/golangci-lint/HEAD/install.sh | sh -s -- -b "$(go env GOPATH)/bin" v2.12.2
      - name: Lint
        run: golangci-lint run ./...
      - name: Format check
        run: golangci-lint fmt --diff ./...

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: false
      - name: Install gotestsum
        run: go install gotest.tools/gotestsum@v1.13.0
      - name: Test
        run: gotestsum --format pkgname -- -coverprofile=coverage.out -covermode=atomic ./...
      - name: Report coverage
        uses: k1LoW/octocov-action@v1
```

（octocov-action は `GITHUB_TOKEN` を自動で使用するため env 追加は不要）

- [ ] **Step 3: YAML がパースできることを確認**

Run: `python3 -c "import yaml,sys; [yaml.safe_load(open(f)) for f in ['.octocov.yml','.github/workflows/ci.yaml']]; print('yaml ok')"`
Expected: `yaml ok`

- [ ] **Step 4: コミット**

```bash
git add .octocov.yml .github/workflows/ci.yaml
git commit -m "feat: add CI workflow with parallel lint and test jobs"
```

- [ ] **Step 5: プッシュして PR を作成し、CI の実挙動を検証**

```bash

gh pr create --fill --base main
```

- 両ジョブが成功する（lint 指摘なし・フォーマット崩れなし・テスト全通過・カバレッジ ≥ 80%）。
- octocov が PR にカバレッジ（約82%）をコメント報告する。
